# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.2.x   | Yes       |
| 0.1.x   | No        |

## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

Please report security issues by emailing: **security@veritasaequitas.com**

> Note: If this email bounces, contact the maintainer directly via the GitHub profile at [@Ap6pack](https://github.com/Ap6pack).

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Fix or mitigation**: Target 30 days for critical issues

## Disclosure

We follow coordinated disclosure. We will:
1. Confirm the issue and determine its impact
2. Develop and test a fix
3. Release the fix and publish an advisory
4. Credit the reporter (unless they prefer anonymity)

We ask that you do not publicly disclose the issue until we have released a fix.
